```typescript
// ClientOpts = { siteId: string, reporterUrl: string, reportOnLocalhost?: boolean, autoTrackPageviews?: boolean }
export function init(opts: ClientOpts): void;
```

Key behavior:

- `reportOnLocalhost` defaults to `false` — localhost tracking is OFF by default
- `autoTrackPageviews` defaults to `true` — monkey-patches `history.pushState` and listens to `popstate`
- SvelteKit uses `history.pushState` for client-side navigation, so auto-tracking works without `afterNavigate`
- The README says `deploymentUrl` but the actual source code uses `reporterUrl` — use `reporterUrl`

**Step 2: Modify `packages/web/src/routes/+layout.svelte`**

Add the Counterscale tracker init inside a `$effect` in the existing `<script lang="ts">` block. The complete file should be:

```svelte
<script lang="ts">
	import '../app.css';
	import favicon from '$lib/assets/favicon.svg';
	import * as Counterscale from '@counterscale/tracker';

	let { children } = $props();

	$effect(() => {
		if (typeof window === 'undefined') return;

		const hostname = window.location.hostname;
		const isProduction = hostname === 'traskriver.com' || hostname === 'www.traskriver.com';

		if (!isProduction) return;

		Counterscale.init({
			siteId: 'traskriver.com',
			reporterUrl: 'https://counterscale.jspn.workers.dev/tracker'
		});

		return () => {
			Counterscale.cleanup();
		};
	});
</script>

<svelte:head>
	<link rel="icon" href={favicon} />
	<title>Trask River Cam</title>
</svelte:head>

{@render children()}
```

**Key decisions implemented (per CONTEXT.md):**

- **D-01:** Programmatic init in `+layout.svelte` inside `$effect` — NOT a script tag in `app.html`
- **D-02:** Uses `reporterUrl` (not `deploymentUrl`) pointing to `https://counterscale.jspn.workers.dev/tracker`
- **D-03:** `@counterscale/tracker` ^3.4.1 is the only new dependency
- **D-04:** Production-only gating via hostname check — only `traskriver.com` and `www.traskriver.com` trigger tracking
- **D-05:** Dev (localhost — already blocked by tracker's `reportOnLocalhost: false` default) AND preview deployments (non-matching hostnames like `*.pages.dev` or `*.workers.dev`) do not pollute the dashboard

**Why hostname check instead of `$app/environment`:**

- `$app/environment` provides `browser` and `building` — neither distinguishes production from preview deployments on Cloudflare
- Hostname check is the most reliable gate: only the exact production domain triggers tracking
- `reportOnLocalhost: false` (tracker default) provides a second layer of protection for dev

**SPA navigation handling:**

- `autoTrackPageviews` defaults to `true` — the tracker monkey-patches `history.pushState` and listens to `popstate`
- SvelteKit uses `history.pushState` for client-side navigation
- Therefore auto-tracking captures SPA navigation without any `afterNavigate` manual calls
- No additional code needed

**Cleanup:**

- The `$effect` returns a cleanup function calling `Counterscale.cleanup()` which restores the original `history.pushState` and removes the `popstate` listener
