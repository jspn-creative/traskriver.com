**1. Remove the remount retry loop (RC1 — lines 245-267):**
Delete the entire `$effect` block that starts `if (!sessionActive || isPlaying) return;` and runs `setInterval(RETRY_INTERVAL_MS=4000)` to increment `playerKey`. This is the nuclear retry strategy that destroys HLS.js's internal recovery.

**2. Remove `playerKey`, `cacheBustedSrc`, and the `{#key playerKey}` wrapper:**

- Delete `let playerKey = $state(0);` (line 38)
- Delete `const RETRY_INTERVAL_MS = 4_000;` (line 40)

**3. Remove the `xhrSetup` Cache-Control header (RC5 — lines 81-83):**
Delete the `xhrSetup` property from the HLS provider config object inside the `onProviderChange` handler. The line is:

```js
xhrSetup: (xhr: XMLHttpRequest) => {
  xhr.setRequestHeader('Cache-Control', 'no-cache, no-store');
}
```

This sets a custom header that forces CORS preflight and is redundant (HLS.js already handles caching for live streams).

**4. Consolidate the dual error handlers into one (RC2 + logging):**
Delete `onLiveError` function (lines 180-217) and its usage as `onerror={onLiveError}` on `<media-player>`.
Delete the `handleError` event listener `$effect` (lines 92-160).

Replace with a single `$effect` that attaches ONE error handler to `hls-error` events. The new handler logic:

```typescript
$effect(() => {
	if (!player) return;
	const el = player;

	const handleHlsError = (e: any) => {
		const detail = e?.detail ?? {};
		const errorType = detail.type ?? '';
		const errorDetails = detail.details ?? '';
		const isFatal = detail.fatal === true;
		const code = detail.response?.code;

		// levelEmptyError (HTTP 200 but no segments) is EXPECTED during
		// Cloudflare Stream warmup. HLS.js retries internally with backoff.
		// Do NOT surface this as an error. Log once at debug level.
		if (errorDetails === 'levelEmptyError') {
			if (__DEV__) console.debug('[VideoPlayer] empty manifest — HLS.js will retry');
			return;
		}

		// manifestParsingError / 204 = stream not yet broadcasting. Expected.
		if (errorDetails === 'manifestParsingError' || code === 204) {
			if (__DEV__) console.debug('[VideoPlayer] manifest not ready — waiting');
			return;
		}

		// Non-fatal errors: HLS.js handles recovery internally.
		// Log at debug level only. No UI state change.
		if (!isFatal) {
			if (__DEV__) console.debug('[VideoPlayer] non-fatal HLS error:', errorDetails);
			return;
		}

		// Fatal error: HLS.js cannot recover. Surface to page.
		console.warn('[VideoPlayer] fatal HLS error:', errorDetails);
		hasError = true;
		isPlaying = false;
		onError?.();
	};

	el.addEventListener('hls-error', handleHlsError);
	return () => el.removeEventListener('hls-error', handleHlsError);
});
```

**Important:** Use `const __DEV__ = import.meta.env.DEV;` at the top of the script (after imports) for the conditional debug logging. This ensures debug logs are tree-shaken in production builds.

**5. Clean up console logging (STRM-04):**

- Replace the `log` and `logErr` helper functions (lines 22-29) that use `console.log` / `console.error` for everything.
- Keep only these meaningful state logs:

  - Remove the `log('defineCustomElements()')` line
  - Remove `log('attach listeners', ...)` and `log('cleanup listeners')` lines
  - Remove `log('starting remount retry loop')` and `log('remounting player (retry)')` lines (these go away with the retry loop deletion anyway)

**6. Keep the `onLivePlaying` handler (line 173-178) as-is** — it correctly sets `isPlaying = true` and calls `onPlaying?.()`. Keep the `onplaying={onLivePlaying}` attribute on `<media-player>`.

**7. Keep the fullscreen logic, poster fallback, and template structure** — only the error handling, retry loop, and logging change.

**8. Keep the provider-change listener** but remove the `xhrSetup` from it. The timeout configs (`fragLoadingTimeOut`, `manifestLoadingTimeOut`, `levelLoadingTimeOut`) and live sync configs (`liveSyncDurationCount`, `liveMaxLatencyDurationCount`) should stay — they're reasonable.

**9. Add a last-resort remount** — if a fatal error occurs AND the session is still active, set a single 10-second timeout (not an interval) that remounts the player ONE time as a final recovery attempt. This preserves the ability to recover from truly broken HLS.js instances while not being the primary strategy:

```typescript
let remountTimeout: ReturnType<typeof setTimeout> | undefined;

$effect(() => {
	// Last-resort: if fatal error + session active, try ONE remount after 10s
	if (!hasError || !sessionActive || isPlaying) {
		clearTimeout(remountTimeout);
		return;
	}

	remountTimeout = setTimeout(() => {
		if (__DEV__) console.debug('[VideoPlayer] last-resort remount after fatal error');
		playerKey += 1;
		hasError = false;
	}, 10_000);

	return () => clearTimeout(remountTimeout);
});
```

For this last-resort remount, keep `playerKey` and the `{#key playerKey}` wrapper, but remove `cacheBustedSrc` — use `liveSrc` directly. The remount itself creates a fresh HLS.js instance which is sufficient.
