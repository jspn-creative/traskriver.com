Output: Modified +page.svelte with button → demand → player flow, unchanged +page.server.ts (NO demand writes on page load).

    **Key behavioral changes (from CONTEXT.md):**
    1. Every page load shows a "Start stream" button — NO video player visible by default
    2. User clicks button → POST to `/api/stream/demand` → video player loads immediately with "Starting stream..." state

    **Implementation approach:**

    Add a new state variable to track demand registration:
    ```typescript
    let demandRegistered = $state(false);
    let demandLoading = $state(false);
    let demandError = $state<string | null>(null);
    ```

    Create a demand registration function:
    ```typescript
    const registerDemand = async () => {
      demandLoading = true;
      demandError = null;
      try {
        const res = await fetch('/api/stream/demand', { method: 'POST' });
        if (!res.ok) {
          throw new Error('Failed to start stream');
        }
        demandRegistered = true;
        // Transition to connecting phase after demand registered
        handleBeginConnection();
      } catch (e) {
        demandError = e instanceof Error ? e.message : 'Failed to start stream';
      } finally {
        demandLoading = false;
      }
    };
    ```

    **Template changes:**

    The existing `handleBeginConnection` is called by `PassDetailsPanel` via `onBeginConnection` prop. We need to:
    1. Gate the `<svelte:boundary>` block (which contains VideoPlayer + getStreamInfo()) behind `demandRegistered`
    2. When `!demandRegistered`, show the background image (defaultJpg) with blur overlay and the "Start stream" button
    3. When `demandRegistered`, show the existing VideoPlayer flow

    Specifically, inside the `<main>` element, wrap the `<svelte:boundary>` content:

    ```svelte
    {#if demandRegistered}
      <svelte:boundary>
        {#snippet pending()}
          <div class="absolute inset-0 z-0 flex items-center justify-center">
            <p class="animate-pulse text-sm text-secondary">Starting stream…</p>
          </div>
        {/snippet}
        {#snippet failed(error, reset)}
          <!-- existing error UI -->
        {/snippet}

        {@const stream = await getStreamInfo()}
        <!-- existing VideoPlayer + overlay markup -->
      </svelte:boundary>
    {:else}
      <!-- Button-first state: background image + blur + start button -->
      <div class="absolute inset-0 z-0 overflow-hidden">
        <img src={defaultJpg} alt="" class="h-full w-full object-cover" />
        <div class="absolute inset-0 bg-linear-to-b from-primary/30 via-light/30 to-primary/30 backdrop-blur-2xl"></div>
      </div>
    {/if}
    ```

    The "Start stream" button should be integrated into the existing `PassDetailsPanel` flow or added as a standalone element. Looking at the current code, `PassDetailsPanel` has an `onBeginConnection` callback that currently triggers the phase transition. The button should call `registerDemand()` instead. Update the `onBeginConnection` prop to wrap demand registration:
