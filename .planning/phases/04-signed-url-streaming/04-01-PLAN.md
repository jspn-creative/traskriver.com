  <action>
Create `scripts/setup-signing.ts` following the exact same structure as `scripts/push-stream.ts`:

1. Read `CF_STREAM_ACCOUNT_ID` and `CF_STREAM_API_TOKEN` from `process.env`.
2. Validate both are present — if either is missing, `console.error('Missing {VAR_NAME} in your environment.')` and `process.exit(1)`.
3. Call the CF Stream signing key creation API:
   ```
   POST https://api.cloudflare.com/client/v4/accounts/{accountId}/stream/keys
   Content-Type: application/json
   Authorization: Bearer {token}
   ```
   Body: `{}` (empty JSON object).
4. Parse response JSON. If `result.success === false` or the fetch fails, log the error and `process.exit(1)`.
5. Extract `result.id` (key ID) and `result.jwk` (base64-encoded JWK string) from the response.
6. Print the following to stdout (exactly — these are the lines the developer pastes into `.env`):

   ```
   Add to your .env:

   CF_STREAM_SIGNING_KEY_ID={result.id}
   CF_STREAM_SIGNING_JWK={result.jwk}
   ```

The script should be plain TypeScript using `fetch` (available in Bun) — no extra dependencies.

Note: The CF API response wraps its result in a `result` key alongside a `success` boolean:

```json
{
	"result": { "id": "abc123", "jwk": "eyJ...", "pem": "..." },
	"success": true,
	"errors": [],
	"messages": []
}
```
