    **Demand poller module** (`packages/relay/src/poller.ts`):
    Create `DemandPoller` class that polls the demand API and tracks consecutive failures.

    ```typescript
    export interface PollResult {
      success: boolean;
      shouldStream: boolean;
      ttlSeconds: number;
      consecutiveFailures: number;
    }

    export class DemandPoller {
      constructor(config: { demandApiUrl: string; bearerToken: string; requestTimeoutMs: number });
      async poll(): Promise<PollResult>;
      resetFailures(): void;
      getConsecutiveFailures(): number;
    }
    ```

    Implementation:
    - `poll()` calls `fetch(config.demandApiUrl, { method: "GET", headers: { "Authorization": "Bearer " + config.bearerToken }, signal: AbortSignal.timeout(config.requestTimeoutMs) })`
    - On success: parse JSON as `DemandResponse`, reset consecutive failures to 0, return `{ success: true, shouldStream: response.shouldStream, ttlSeconds: response.ttlSeconds, consecutiveFailures: 0 }`
    - On failure (network error, timeout, non-200 status): increment consecutive failures, log error via `log.error()`, return `{ success: false, shouldStream: false, ttlSeconds: 0, consecutiveFailures: this.failures }`
    - `AbortSignal.timeout(config.requestTimeoutMs)` for the 8s timeout — Bun supports this natively
    - Log each poll result: `log.info("poll: shouldStream={bool}, ttl={N}s")` on success, `log.error("poll failed: {reason} (failures: {N})")` on error
    - `resetFailures()` sets consecutive failure counter to 0 (called when entering idle after safety stop)
