```ts
export const ConfigSchema = z.object({
	NODE_ENV: z.enum(['development', 'production', 'test']).default('production'),
	LOG_LEVEL: z.enum(['trace', 'debug', 'info', 'warn', 'error', 'fatal']).default('info'),
	PORT: z.coerce.number().int().positive().default(8080),
	RTSP_URL: z.string().url(),

	HLS_DIR: z.string().default('/run/stream/hls'),
	MEDIAMTX_BIN: z.string().default('mediamtx')
});

export type Config = z.infer<typeof ConfigSchema>;

export function loadConfig() {
	const result = ConfigSchema.safeParse(process.env);
	if (!result.success) {
		process.stderr.write(`FATAL: invalid env:\n${z.prettifyError(result.error)}\n`);
		process.exit(1);
	}
	return result.data;
}
```

Critical key notes (do NOT regress to obsolete syntax):

- `rtspTransport: tcp` (per-path) — NOT `sourceProtocol: tcp` (pre-v1.3 obsolete; CONTEXT.md uses old wording, RESEARCH Pitfall #1 corrects it).
- `hlsAlwaysRemux: yes` is REQUIRED — without it the muxer is lazy and first viewer waits 2-5s for cold start (RESEARCH Pitfall #2). Locks in 24/7 readiness.
- `hlsVariant: mpegts` gives `.ts` segments (passthrough H.264, no transcode). `fmp4` would also be passthrough but criterion #4 says `.ts`.
- `hlsAllowOrigins: ['*']` — required so the Phase 9 web client (different origin) can fetch the manifest.
- `rtsp: no` — we run AS an RTSP client (puller), not as a server. Disabling the server side saves the port + attack surface.
- `logLevel: info` — never `debug` (RESEARCH §9 — debug logs leak full RTSP URL even though MediaMTX redacts at info).
- All values come from env (zod-validated). No string interpolation hazards because zod constrains each input type.
- Trailing newline after the closing backtick is intentional — yaml parsers tolerate but file editors complain without it.
- No explicit return type (user rule).

This file is pure: takes a `Config`, returns a string. No side effects, no fs access — Plan 02 owns `writeFile`.

**`packages/stream/src/mediamtx-api.ts`** (verbatim from RESEARCH §Code Examples):

```ts
export type PathInfo = {
	ready: boolean;
	bytesReceived: number;
	codec: string | null;
};

const VIDEO_CODECS = new Set([
	'AV1',
	'VP9',
	'VP8',
	'H265',
	'H264',
	'MPEG-4 Video',
	'MPEG-1/2 Video',
	'M-JPEG'
]);

export async function getPathInfo(apiPort: number, path: string, timeoutMs = 2_000) {
	const url = `http://127.0.0.1:${apiPort}/v3/paths/get/${encodeURIComponent(path)}`;
	const res = await fetch(url, { signal: AbortSignal.timeout(timeoutMs) });
	if (!res.ok) throw new Error(`mediamtx api ${res.status}`);

	const body = (await res.json()) as {
		ready?: boolean;
		bytesReceived?: number;
		tracks?: string[];
		tracks2?: Array<{ codec: string }>;
	};

	const codecs = body.tracks2?.map((t) => t.codec) ?? body.tracks ?? [];
	const videoCodec = codecs.find((c) => VIDEO_CODECS.has(c)) ?? null;

	return {
		ready: body.ready === true,
		bytesReceived: typeof body.bytesReceived === 'number' ? body.bytesReceived : 0,
		codec: videoCodec
	} satisfies PathInfo;
}
```

Notes:

- Native `fetch` (Node 22 built-in). Do NOT add `undici` / `got` / `axios` (RESEARCH §Don't Hand-Roll).
- `AbortSignal.timeout(timeoutMs)` is mandatory (Pitfall #5) — fetch hangs forever if MediaMTX API goroutine starves.
- Bound to `127.0.0.1` explicitly — same-host poller, no auth needed (MediaMTX `authInternalUsers` allows loopback by default; RESEARCH §2 Auth note).
- `encodeURIComponent(path)` defensive — path is a config-controlled string.
- Codec extraction prefers `tracks2[].codec` (v1.17+) and falls back to `tracks[]` (deprecated but still emitted).
- `VIDEO_CODECS` filter strips audio (`Opus`, `MPEG-4 Audio`) so the supervisor compares against video only.
- Codec value `'H264'` is the exact PascalCase string MediaMTX emits (RESEARCH §11) — Plan 02's strict equality check `codec === 'H264'` matches this.
- No retry loop here — caller (Plan 02 supervisor) handles retry semantics on poll failure.
- No explicit return type on `getPathInfo` (user rule); inner `satisfies PathInfo` keeps the shape locked without a declared return.
