In `packages/web/src/lib/components/PassDetailsPanel.svelte`, change the `buttonDisabled` derivation on line 28 from:

```typescript
let buttonDisabled = $derived(sessionActive || demandLoading);
```

to:

```typescript
let buttonDisabled = $derived((sessionActive && phase !== 'unavailable') || demandLoading);
```

This keeps the button disabled during active stream phases (starting, live, viewing, ended_confirming) and during demand loading, but allows it in the `unavailable` phase where users should be able to retry.

Additionally, update the button's visual styling so the unavailable state gets the clickable appearance instead of the disabled appearance. On line 91-98, the button class conditional already handles `isStarting` (which includes `unavailable`) — but the `buttonDisabled` ternary on lines 91-92 applies `cursor-not-allowed shadow-inner` when disabled. Since unavailable is no longer disabled, the button will correctly pick up the non-disabled branch (`cursor-pointer bg-primary hover:bg-primary/90 active:scale-[0.98]`). However, the `isStarting` check on line 93-95 applies `bg-secondary/90` which overrides `bg-primary`. This is acceptable — the unavailable CTA will show the muted secondary background with the spinner icon and "Try starting stream" label, which is visually appropriate for a tentative retry action.

No other changes needed. The `sessionActive` variable is used in `+page.svelte` for header auto-hide and video overlay logic — it should NOT be modified here. The fix is scoped entirely to the `buttonDisabled` derivation within PassDetailsPanel.
