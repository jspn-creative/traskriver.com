Create `packages/relay/scripts/configure.ts` — an idempotent Bun script for app-level deployment. This is the script that GitHub Actions (and operators manually) run to deploy updates to the relay.

Per D-03 and D-12, the script does: git pull, bun install, copy systemd units, daemon-reload, restart service.

```typescript
#!/usr/bin/env bun
/**
 * configure.ts — Idempotent relay configuration/deploy script.
 *
 * Run on the Pi to apply code and config updates:
 *   sudo bun run packages/relay/scripts/configure.ts
 *
 * Safe to re-run at any time. Called by:
 *   - GitHub Actions deploy workflow (on push to main)
 *   - Operator manually via Tailscale SSH
 */

const RELAY_DIR = "/opt/river-relay";
const SYSTEMD_DIR = "/etc/systemd/system";
const CONFIG_DIR = `${RELAY_DIR}/packages/relay/config`;

const UNITS = [
  "river-relay.service",
  "river-relay-reset.timer",
  "river-relay-reset.service",
] as const;

function run(cmd: string[], opts?: { cwd?: string }): boolean {
  const result = Bun.spawnSync(cmd, {
    cwd: opts?.cwd ?? RELAY_DIR,
    stdout: "inherit",
    stderr: "inherit",
  });
  return result.success;
}

function log(msg: string) {
  console.log(`[configure] ${msg}`);
}

// ─── Preflight ─────────────────────────────────────────────
if (process.getuid?.() !== 0) {
  console.error("ERROR: configure.ts must be run as root (use sudo)");
  process.exit(1);
}

if (!await Bun.file(`${RELAY_DIR}/.env`).exists()) {
  console.error("ERROR: /opt/river-relay/.env not found. Run setup.sh first.");
  process.exit(1);
}

// ─── 1. Pull latest code ──────────────────────────────────
log("Pulling latest code...");
if (!run(["git", "pull", "--ff-only"])) {
  console.error("ERROR: git pull failed (merge conflict or network issue)");
  process.exit(1);
}

// ─── 2. Install dependencies (only when lockfile changed) ─
const previousLock = Bun.spawnSync(["git", "rev-parse", "HEAD@{1}"], {
  cwd: RELAY_DIR,
  stdout: "pipe",
  stderr: "ignore",
});
const previousRef = new TextDecoder().decode(previousLock.stdout).trim();
const lockChanged = previousRef
  ? run(["git", "diff", "--quiet", `${previousRef}..HEAD`, "--", "bun.lock"]) === false
  : true;

if (lockChanged) {
  log("bun.lock changed, installing dependencies...");
  if (!run(["bun", "install", "--frozen-lockfile"])) {
    log("Retrying bun install without --frozen-lockfile...");
    if (!run(["bun", "install"])) {
      console.error("ERROR: bun install failed");
      process.exit(1);
    }
  }
} else {
  log("bun.lock unchanged, skipping bun install");
}

// ─── 3. Copy systemd units if changed ─────────────────────
let unitsChanged = false;

for (const unit of UNITS) {
  const src = `${CONFIG_DIR}/${unit}`;
  const dst = `${SYSTEMD_DIR}/${unit}`;

  const srcFile = Bun.file(src);
  const dstFile = Bun.file(dst);

  if (!await srcFile.exists()) {
    console.error(`ERROR: ${src} not found`);
    process.exit(1);
  }

  const srcContent = await srcFile.text();
  const dstContent = await dstFile.exists() ? await dstFile.text() : "";

  if (srcContent !== dstContent) {
    log(`Updating ${unit}...`);
    await Bun.write(dst, srcContent);
    unitsChanged = true;
  } else {
    log(`${unit} unchanged, skipping`);
  }
}

// ─── 4. Reload systemd if units changed ───────────────────
if (unitsChanged) {
  log("Reloading systemd daemon...");
  run(["systemctl", "daemon-reload"]);
}

// ─── 5. Restart relay service ─────────────────────────────
log("Restarting river-relay service...");
if (!run(["systemctl", "restart", "river-relay.service"])) {
  console.error("ERROR: service restart failed. Rolling back to previous commit...");
  run(["git", "reset", "--hard", "HEAD@{1}"]);
  run(["systemctl", "restart", "river-relay.service"]);
  process.exit(1);
}

// Ensure timer is enabled and running
run(["systemctl", "enable", "river-relay-reset.timer"]);
run(["systemctl", "start", "river-relay-reset.timer"]);

// ─── 6. Verify (systemd + /health) ───────────────────────
log("Verifying service status...");
const status = Bun.spawnSync(["systemctl", "is-active", "river-relay.service"], {
  stdout: "pipe",
});
const state = new TextDecoder().decode(status.stdout).trim();

if (state === "active" || state === "activating") {
  log(`Service is ${state} ✓`);
} else {
  log(`WARNING: Service state is '${state}' — check: journalctl -u river-relay -n 50`);
}

const health = Bun.spawnSync(
  ["curl", "-fsS", "--max-time", "3", "http://127.0.0.1:9090/health"],
  { stdout: "pipe", stderr: "pipe" }
);
if (health.success) {
  log("Health endpoint check passed ✓");
} else {
  log("WARNING: /health check failed (service may still be warming up or health disabled)");
}

log("Configure complete");
```

Key implementation details:
- Uses `Bun.spawnSync` for subprocess control (matches relay codebase pattern of Bun.spawn)
- `--ff-only` on git pull prevents merge commits; fails cleanly if there's a divergence
- Checks whether `bun.lock` changed and skips `bun install` on code-only deploys
- `--frozen-lockfile` on bun install first; falls back to regular install if lockfile changed
- Compares file contents before copying systemd units (avoids unnecessary daemon-reload)
- Only does daemon-reload if a unit file actually changed
- Verifies service state via systemd and `/health` endpoint after restart
- On restart failure, attempts immediate rollback to `HEAD@{1}` and restarts previous revision
- Root check at top — must run with sudo
- .env check — fails early if setup.sh hasn't been run

**`.github/workflows/deploy-relay.yml`** — GitHub Actions workflow triggered on push to main, path-filtered to relay and shared packages (per D-10, D-11):

```yaml
name: Deploy Relay

on:
  push:
    branches: [main]
    paths:
      - "packages/relay/**"
      - "packages/shared/**"

jobs:
  deploy:
    name: Deploy to Pi
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - name: Connect to Tailscale
        uses: tailscale/github-action@v3
        with:
          oauth-client-id: ${{ secrets.TS_OAUTH_CLIENT_ID }}
          oauth-secret: ${{ secrets.TS_OAUTH_SECRET }}
          tags: tag:ci

      - name: Deploy via SSH
        env:
          RELAY_HOST: ${{ secrets.RELAY_TAILSCALE_HOSTNAME }}
        run: |
          ssh -o StrictHostKeyChecking=accept-new \
              -o ConnectTimeout=30 \
              root@"$RELAY_HOST" \
              "cd /opt/river-relay && bun run packages/relay/scripts/configure.ts"
```

Key implementation details:
- `tailscale/github-action@v3` — official Tailscale GitHub Action; connects the runner to Tailnet
- Uses OAuth client credentials (not auth key) — recommended for CI/CD by Tailscale docs
- `tags: tag:ci` — Tailscale ACL tag for CI runners; must be pre-configured in Tailscale admin
- `StrictHostKeyChecking=accept-new` — accepts host key on first connect, rejects changes after
- `ConnectTimeout=30` — fails fast if Pi is unreachable
- SSH as root because configure.ts needs sudo for systemctl operations
- Path filter: only `packages/relay/**` and `packages/shared/**` — web-only changes don't trigger deploy (per D-10)
- `timeout-minutes: 10` — prevents stuck jobs from burning minutes

**Required GitHub secrets (document in README):**
- `TS_OAUTH_CLIENT_ID` — Tailscale OAuth client ID (create at https://login.tailscale.com/admin/settings/oauth)
- `TS_OAUTH_SECRET` — Tailscale OAuth client secret
- `RELAY_TAILSCALE_HOSTNAME` — Pi's Tailscale MagicDNS hostname (e.g., `pi-relay.tailnet-name.ts.net`)

TypeScript relay service that polls for viewer demand and streams RTSP to Cloudflare Stream via ffmpeg.

## Initial Setup (One-Time)

1. Flash Pi OS Lite to SD card
2. Copy `.env` to the boot partition (FAT32):
   ```
   cp .env /Volumes/bootfs/.env   # macOS
   cp .env /media/$USER/bootfs/.env  # Linux
   ```
   See `.env.example` for required values (STREAM_URL, RTSP_URL, DEMAND_API_URL, STATUS_API_URL, RELAY_BEARER_TOKEN, TAILSCALE_AUTHKEY).
3. Boot Pi, SSH in (default pi user), run:
   ```bash
   sudo bash /boot/firmware/setup.sh  # or download from repo
   ```
   Or clone the repo first and run:
   ```bash
   git clone https://github.com/your-user/river-stream.git /tmp/river-stream
   sudo bash /tmp/river-stream/packages/relay/scripts/setup.sh
   ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions → Pi SSH | CI runner executes commands on relay device over Tailscale |
| Tailscale OAuth → Tailnet | CI authenticates to Tailscale using OAuth credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Spoofing | GitHub Actions SSH | mitigate | Use Tailscale OAuth (not auth keys); `tag:ci` ACL restricts what CI can access; SSH to specific hostname only |
| T-09-07 | Tampering | git pull on Pi | mitigate | `--ff-only` rejects non-fast-forward merges; only main branch deployed; repo access controlled by GitHub |
| T-09-08 | Information Disclosure | GitHub secrets | mitigate | TS_OAUTH_CLIENT_ID, TS_OAUTH_SECRET, RELAY_TAILSCALE_HOSTNAME stored as GitHub encrypted secrets; never logged |
| T-09-09 | Denial of Service | Deploy job stuck | mitigate | `timeout-minutes: 10` kills stuck jobs; `ConnectTimeout=30` on SSH prevents indefinite hang |
| T-09-10 | Elevation of Privilege | SSH as root | accept | configure.ts needs root for systemctl; Pi is single-purpose device with Tailscale ACL controlling access; acceptable for single-device deployment |
</threat_model>
