**`packages/relay/config/river-relay.service`:**
```ini
[Unit]
Description=River Stream Relay Service
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=relay
Group=relay
WorkingDirectory=/opt/river-relay/packages/relay
ExecStart=/usr/local/bin/bun run src/index.ts
Restart=always
RestartSec=5
StartLimitBurst=10
StartLimitIntervalSec=300

EnvironmentFile=/opt/river-relay/.env

# Security hardening (per D-17)
NoNewPrivileges=true
ProtectSystem=strict
ReadWritePaths=/opt/river-relay
PrivateTmp=true

# Logging
StandardOutput=journal
StandardError=journal
SyslogIdentifier=river-relay

[Install]
WantedBy=multi-user.target
```

Key values explained:
- `StartLimitBurst=10` + `StartLimitIntervalSec=300` = max 10 restarts per 5 minutes before systemd stops trying (per D-16; relay's own 15s cooldown means max ~20 attempts/5min if crashes are instant, but with RestartSec=5 the practical max is about 15/5min — 10 is a safe limit)
- `After=network-online.target` ensures network is up before relay tries to poll demand API
- `WorkingDirectory` is within ReadWritePaths, so bun can write temp/cache files
- Bun path is `/usr/local/bin/bun` which is where the official Bun installer places it

**`packages/relay/config/river-relay-reset.timer`:**
```ini
[Unit]
Description=Reset River Relay failure counter every 15 minutes

[Timer]
OnBootSec=15min
OnUnitActiveSec=15min
Unit=river-relay-reset.service

[Install]
WantedBy=timers.target
```

**`packages/relay/config/river-relay-reset.service`:**
```ini
[Unit]
Description=Reset River Relay systemd failure counter

[Service]
Type=oneshot
ExecStart=/usr/bin/systemctl reset-failed river-relay.service
```

The timer fires every 15 minutes (per D-16/DEPL-03) and runs the oneshot that resets the failure counter. This ensures the relay always eventually recovers after transient failure bursts — even if systemd hits StartLimitBurst, the counter resets and the service can restart again.

**`packages/relay/scripts/setup.sh`** — One-time Pi provisioning script. Must be run as root on a fresh Pi OS Lite. The script is idempotent where possible (checks before acting) but intended for first-boot use.

```bash
#!/usr/bin/env bash
set -euo pipefail

# ─── Preflight ──────────────────────────────────────────────────
if [[ $EUID -ne 0 ]]; then
  echo "ERROR: setup.sh must be run as root (use sudo)"
  exit 1
fi

RELAY_DIR="/opt/river-relay"
RELAY_USER="relay"
REPO_URL="https://github.com/your-user/river-stream.git"

# ─── 1. Locate and move .env from boot partition (per D-06) ────
# Pi OS Lite mounts boot at /boot/firmware (Bookworm+) or /boot (legacy)
ENV_SRC=""
if [[ -f /boot/firmware/.env ]]; then
  ENV_SRC="/boot/firmware/.env"
elif [[ -f /boot/.env ]]; then
  ENV_SRC="/boot/.env"
fi

if [[ -z "$ENV_SRC" ]]; then
  echo "ERROR: No .env found on boot partition (/boot/firmware/.env or /boot/.env)"
  echo "Place .env on the FAT32 boot partition before running setup.sh"
  exit 1
fi

echo "Found .env at $ENV_SRC"

# Source .env for TAILSCALE_AUTHKEY (needed during setup)
# shellcheck disable=SC1090
source "$ENV_SRC"

if [[ -z "${TAILSCALE_AUTHKEY:-}" ]]; then
  echo "ERROR: TAILSCALE_AUTHKEY not found in .env"
  exit 1
fi

# ─── 2. System packages ────────────────────────────────────────
echo "Updating package lists..."
apt-get update -qq

echo "Installing ffmpeg and prerequisites..."
apt-get install -y -qq ffmpeg curl git

# ─── 3. Install Bun ────────────────────────────────────────────
if ! command -v bun &>/dev/null; then
  echo "Installing Bun..."
  curl -fsSL https://bun.sh/install | bash
  # Bun installer puts binary at ~/.bun/bin/bun for current user (root)
  # Move to system-wide location
  cp /root/.bun/bin/bun /usr/local/bin/bun
  chmod +x /usr/local/bin/bun
fi
echo "Bun: $(bun --version)"

# ─── 4. Install and configure Tailscale (per D-09) ─────────────
if ! command -v tailscale &>/dev/null; then
  echo "Installing Tailscale..."
  curl -fsSL https://tailscale.com/install.sh | sh
fi

echo "Authenticating Tailscale and enabling exit node..."
tailscale up --authkey="$TAILSCALE_AUTHKEY" --advertise-exit-node --ssh

# ─── 5. Create relay user (per D-02, D-05) ─────────────────────
if ! id "$RELAY_USER" &>/dev/null; then
  echo "Creating $RELAY_USER user..."
  useradd --system --shell /usr/sbin/nologin --home-dir "$RELAY_DIR" "$RELAY_USER"
fi

# ─── 6. Clone repo and set ownership (per D-05) ────────────────
if [[ ! -d "$RELAY_DIR/.git" ]]; then
  echo "Cloning repository to $RELAY_DIR..."
  git clone "$REPO_URL" "$RELAY_DIR"
fi
chown -R "$RELAY_USER:$RELAY_USER" "$RELAY_DIR"

# ─── 7. Move .env to relay directory (per D-06) ────────────────
echo "Moving .env to $RELAY_DIR/.env..."
cp "$ENV_SRC" "$RELAY_DIR/.env"
chmod 600 "$RELAY_DIR/.env"
chown "$RELAY_USER:$RELAY_USER" "$RELAY_DIR/.env"
rm -f "$ENV_SRC"
echo "Boot partition .env deleted"

# ─── 8. SD card hardening (per D-04) ───────────────────────────
echo "Hardening SD card..."

# Disable swap
dphys-swapfile swapoff 2>/dev/null || true
dphys-swapfile uninstall 2>/dev/null || true
systemctl disable dphys-swapfile 2>/dev/null || true
echo "Swap disabled"

# Volatile journald (Storage=volatile)
mkdir -p /etc/systemd/journald.conf.d
cat > /etc/systemd/journald.conf.d/volatile.conf <<'JOURNALD'
[Journal]
Storage=volatile
RuntimeMaxUse=32M
JOURNALD
systemctl restart systemd-journald
echo "Journald set to volatile (RAM only, 32M max)"

# tmpfs for /tmp and /var/tmp
if ! grep -q "tmpfs /tmp" /etc/fstab; then
  echo "tmpfs /tmp tmpfs defaults,noatime,nosuid,nodev,size=64M 0 0" >> /etc/fstab
fi
if ! grep -q "tmpfs /var/tmp" /etc/fstab; then
  echo "tmpfs /var/tmp tmpfs defaults,noatime,nosuid,nodev,size=32M 0 0" >> /etc/fstab
fi

# noatime on root filesystem
if grep -q " / " /etc/fstab && ! grep -q "noatime" /etc/fstab; then
  sed -i 's|\( / .*defaults\)|\1,noatime|' /etc/fstab
fi
echo "tmpfs and noatime configured (effective after reboot)"

# NOTE: unattended-upgrades explicitly NOT installed (per D-04)

# ─── 9. Install systemd units (per D-18) ───────────────────────
echo "Installing systemd service and timer..."
cp "$RELAY_DIR/packages/relay/config/river-relay.service" /etc/systemd/system/
cp "$RELAY_DIR/packages/relay/config/river-relay-reset.service" /etc/systemd/system/
cp "$RELAY_DIR/packages/relay/config/river-relay-reset.timer" /etc/systemd/system/
systemctl daemon-reload
systemctl enable river-relay.service
systemctl enable river-relay-reset.timer

# ─── 10. Install app dependencies ──────────────────────────────
echo "Installing relay dependencies..."
cd "$RELAY_DIR"
sudo -u "$RELAY_USER" bun install

# ─── 11. Start services ────────────────────────────────────────
echo "Starting relay service..."
systemctl start river-relay-reset.timer
systemctl start river-relay.service

echo ""
echo "═══════════════════════════════════════════════"
echo "  Setup complete!"
echo "═══════════════════════════════════════════════"
echo ""
echo "  Relay service: systemctl status river-relay"
echo "  Relay logs:    journalctl -u river-relay -f"
echo "  Tailscale:     tailscale status"
echo ""
```

**Key implementation details:**
- Root check at top — script must run with sudo
- .env sourced early to get TAILSCALE_AUTHKEY before Tailscale setup
- Bun installed to `/usr/local/bin/bun` (system-wide, not user-local) so systemd can find it
- `tailscale up --ssh` enables Tailscale SSH — allows remote access without configuring OpenSSH keys
- `--advertise-exit-node` per D-09 for routing traffic through Pi to camera network
- relay user has `--system --shell /usr/sbin/nologin` — no interactive login, no sudo
- SD hardening checks before appending to fstab to be idempotent
- Systemd units enabled (start on boot) AND started (start now)
- REPO_URL must be updated to actual repository URL — leave as placeholder with a clear comment

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Boot partition → relay .env | Secrets move from unencrypted FAT32 to chmod 600 file |
| GitHub → Pi (SSH over Tailscale) | Deploy pipeline executes commands on relay device |
| Pi → Demand API | Bearer token auth for relay-to-worker communication |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | .env on boot partition | mitigate | setup.sh copies to /opt/river-relay/.env with chmod 600, deletes boot copy immediately (D-06) |
| T-09-02 | Elevation of Privilege | relay user | mitigate | useradd --system --shell /usr/sbin/nologin; systemd NoNewPrivileges=true, ProtectSystem=strict (D-02, D-17) |
| T-09-03 | Tampering | systemd service file | mitigate | ProtectSystem=strict limits write access; service file owned by root, relay user cannot modify it |
| T-09-04 | Denial of Service | SD card wear | mitigate | Volatile journald, tmpfs, noatime, no swap — reduces writes ~95% (D-04) |
| T-09-05 | Information Disclosure | Tailscale exit node | accept | Exit node is intentional for camera network access (D-09); Tailscale ACLs managed externally |
</threat_model>
