**4. `packages/counterscale-proxy/src/index.ts`:**
The Worker handles two request types:

- **OPTIONS preflight:** Return 204 with CORS headers immediately (no upstream call). Headers:
  - `Access-Control-Allow-Origin: *` (per D-01)
  - `Access-Control-Allow-Methods: GET, POST, OPTIONS`
  - `Access-Control-Allow-Headers: Content-Type`
  - `Access-Control-Max-Age: 86400`

- **POST (and GET) tracker requests:** Forward the request to `https://counterscale.jspn.workers.dev/tracker` preserving method, headers, and body. Clone the upstream response, add CORS headers to the clone, return it.

- **All other methods:** Return 405 Method Not Allowed.

- **Security hardening:** The Worker MUST only proxy to the exact upstream URL `https://counterscale.jspn.workers.dev/tracker`. Do NOT accept an arbitrary destination URL from the request. The upstream URL is hardcoded as a constant `UPSTREAM_URL`. The request path is ignored — all requests proxy to the same upstream regardless of path. This prevents the Worker from being used as an open relay.

Implementation:

```typescript
const UPSTREAM_URL = 'https://counterscale.jspn.workers.dev/tracker';

const CORS_HEADERS: Record<string, string> = {
	'Access-Control-Allow-Origin': '*',
	'Access-Control-Allow-Methods': 'GET, POST, OPTIONS',
	'Access-Control-Allow-Headers': 'Content-Type',
	'Access-Control-Max-Age': '86400'
};

export default {
	async fetch(request: Request): Promise<Response> {
		if (request.method === 'OPTIONS') {
			return new Response(null, { status: 204, headers: CORS_HEADERS });
		}

		if (request.method !== 'GET' && request.method !== 'POST') {
			return new Response('Method Not Allowed', { status: 405 });
		}

		const upstreamResponse = await fetch(UPSTREAM_URL, {
			method: request.method,
			headers: request.headers,
			body: request.method === 'POST' ? request.body : undefined
		});

		const response = new Response(upstreamResponse.body, {
			status: upstreamResponse.status,
			statusText: upstreamResponse.statusText,
			headers: upstreamResponse.headers
		});

		for (const [key, value] of Object.entries(CORS_HEADERS)) {
			response.headers.set(key, value);
		}

		return response;
	}
};
```

<threat_model>

## Trust Boundaries

| Boundary                             | Description                                      |
| ------------------------------------ | ------------------------------------------------ |
| Browser → Proxy Worker               | Untrusted requests from any origin hit the proxy |
| Proxy Worker → Upstream Counterscale | Proxy forwards to a fixed upstream URL           |

## STRIDE Threat Register

| Threat ID | Category               | Component               | Disposition | Mitigation Plan                                                                                                                                                                                        |
| --------- | ---------------------- | ----------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| T-02-01   | Spoofing               | Proxy Worker            | accept      | Public analytics endpoint — no auth needed. Counterscale itself is public. Spoofed pageviews are low-value (analytics, not billing).                                                                   |
| T-02-02   | Tampering              | Request body forwarding | accept      | Proxy passes body as-is to upstream. Upstream validates its own input. No user PII at risk.                                                                                                            |
| T-02-03   | Repudiation            | Analytics data          | accept      | Analytics inherently non-repudiable — pageviews are anonymous fire-and-forget. No audit requirement.                                                                                                   |
| T-02-04   | Information Disclosure | CORS wildcard `*`       | accept      | No cookies, no auth tokens, no PII flows through this proxy. Wildcard is safe per D-01 rationale.                                                                                                      |
| T-02-05   | Denial of Service      | Proxy Worker            | accept      | Cloudflare Workers have built-in DDoS protection and rate limiting. No additional mitigation needed for a low-value analytics proxy.                                                                   |
| T-02-06   | Elevation of Privilege | Open relay abuse        | mitigate    | Upstream URL is hardcoded as `UPSTREAM_URL` constant — the proxy ONLY forwards to `counterscale.jspn.workers.dev/tracker`. Request path is ignored. Cannot be used to proxy to arbitrary destinations. |

</threat_model>
